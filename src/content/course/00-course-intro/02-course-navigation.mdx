# Навигация по курсу

Этот урок поможет вам быстро освоиться с платформой и начать обучение эффективно.

---

## Боковое меню (Sidebar)

Слева на экране находится **боковое меню** — ваш главный навигатор по курсу.

**Что вы найдёте в меню:**

- **Модули** — основные разделы курса (от M00 до M18)
- **Уроки** — темы внутри каждого модуля
- **Прогресс** — сколько уроков вы уже прошли в каждом модуле

**Как пользоваться:**

1. Кликните на **название модуля**, чтобы развернуть список уроков
2. Кликните на **урок**, чтобы перейти к нему
3. Счётчик справа от модуля показывает прогресс (например, `3/6`)

На мобильных устройствах меню скрыто — используйте иконку ☰ в верхнем левом углу.

## Типы контента

Каждый модуль содержит несколько типов учебных материалов:

| Тип | Описание | Как выглядит |
|-----|----------|--------------|
| **Урок** | Теоретический материал с примерами кода и диаграммами | Текст с блоками кода |
| **Квиз** | Вопросы по уроку (multiple-choice, true/false) | Интерактивные карточки |
| **Code Challenge** | PySpark-задача с автоматической проверкой | Редактор кода с кнопкой Run |
| **Экзамен** | Итоговый тест по модулю (10-12 вопросов) | Таймер + вопросы |
| **Docker Lab** | Практическая работа на реальном Spark-кластере | Инструкции + терминал |

## Code Challenges — как это работает

Code challenges используют **spark_sim** — pandas-based эмулятор PySpark API. Вы пишете настоящий PySpark-код, который выполняется в браузере:

1. Прочитайте условие задачи
2. Заполните `# TODO` секции в starter code
3. Нажмите **Run** для выполнения
4. Автоматические тесты проверят ваш вывод
5. После решения прочитайте **Explanation** — там объяснение подхода

> **Совет:** Если вы застряли, нажмите **Show Solution** — но сначала попробуйте сами!

## Рекомендуемый путь обучения

Курс спроектирован для **последовательного прохождения** — каждый модуль опирается на знания из предыдущих:

**Основной путь:** M01 → M02 → M03 → M04 → M05 → ... → M16

**Если вы уже знакомы с основами Spark:**
- Пропустите M01-M03, начните с M04 (Performance Tuning)
- M05 (AQE) и M08 (Monitoring) — ключевые для production

**Если вас интересует только streaming:**
- M09 (Structured Streaming) → M10 (Lakehouse) → M11 (Arrow)
- Предварительно пройдите M01 для понимания архитектуры

## Поиск по курсу

Для быстрого поиска используйте **глобальный поиск**:

- **macOS**: `⌘ + K`
- **Windows/Linux**: `Ctrl + K`

Поиск находит названия уроков, ключевые слова, фрагменты кода и термины.

| Клавиша | Действие |
|---------|----------|
| `↑` / `↓` | Навигация по результатам |
| `Enter` | Перейти к результату |
| `Escape` | Закрыть поиск |

---

## Навигация между уроками

В конце каждого урока есть кнопки:

- **← Предыдущий урок** — вернуться назад
- **Следующий урок →** — перейти к следующей теме

Также вы можете перейти к любому уроку через боковое меню.
